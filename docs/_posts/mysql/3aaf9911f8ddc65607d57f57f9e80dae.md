    <br />      Myisam   <br />           默认的mysql数据库存储引擎, 创建是创建三个文件 .frm(存储表定义), .myd(存储数据), .myi(存储索引), 数据文件和索引文件可以放置在不同的目录,平均分配io, 获取更快的速度, web,数据仓储和其他应用环境下最常用的引擎 <br />      InnoDb <br />           具有,提交, 回滚, 崩溃回复能力 的事务安全, 但是对比myisam innoDb写的处理效果差些并且会占用更多的空间来存放数据和索引, 包括acid事务支持 <br />      Memory <br />           将数据保存在 Ram 中, 需要快速查找引用和其他类似数据环境下, 提供极快的访问 <br />      Merge <br />           允许mysqlDBA或开发人员将一系列的等同的MyISAM表以逻辑方式组合在一起, 并作为一个对象引用他们.对于数据仓储等VLDB环境十分适合 <br />3. 选择合适的数据类型<br />      数据表的存储引擎和数据列 <br />           MyISAM <br />                最好使用固定长度的数据列来代替可变的数据列 <br />           Memory           <br />                目前都是用固定的数据行存储 <br />           InnoDB <br />                建议使用varchar, 内部的存储格式没有区分char和varchar, 主要的性能因素是数据行使用的存储总量, 所以使用varchar来降低空间就是对性能好的优化 <br />4. 数据值的对比<br />      char 和 varchar <br />           1.存储空间的对比 <br />             ![](https://file.wulicode.com/yuque/202208/24/23/16400XiGNqOE.png?x-oss-process=image/resize,h_191)<br />           例如最后一种情况, 如果数据运行在严格模式, 超出规定的值不会保存,而是报错 <br />           2. 搜索的时候varchar不会忽略存储的空格,但是char可以忽略存储的空格. <br />             ![](https://file.wulicode.com/yuque/202208/24/23/1640k6pQN9n1.png?x-oss-process=image/resize,h_92)<br />      text和blog <br />                    1. blob 和 text 在执行大量删除和更新后会产生碎片,建议定期使用optimize table 功能对其进行碎片整理<br />           2. 使用合成的索引 <br />           3. 避免检索大型的blob或text索引 <br />           4. 把blob 或text 列分离到单独的表中 <br />      浮点数和定点数 <br />           插入的数值,如果是浮点数, 可能会造成不确定性 <br />           数据库设计 <br />             ![](https://file.wulicode.com/yuque/202208/24/23/1641LVxNOtA6.png?x-oss-process=image/resize,h_40)<br />           执行语句 <br />             ![](https://file.wulicode.com/yuque/202208/24/23/1641EAAwtavr.png?x-oss-process=image/resize,h_16)<br />           结果是 <br />             ![](https://file.wulicode.com/yuque/202208/24/23/1641PBbfMmVz.png?x-oss-process=image/resize,h_53)<br />           在mysql中 float, double是浮点数, decimal是定点数

5. 字符集的选择<br />      显示字符集 <br />                SHOW CHARACTER SET <br />      字符集和校对规则 <br />           字符集规定了myusql存储字串的方式,校对规则定义了比较字串的方式, 每个字符集至少对应一个校对规则 <br />            SHOW COLLATION LIKE 'utf8%'   显示utf8 的校对规则 <br />      怎样选择合适的字符集 <br />           建议在满足应用的前提下,尽量使用小的字符集, 因为小的字符集意味着能够节省空间,减少网络传输字节数,同时由于存储空间的较小间接的提高了系统的性能           <br />      mysql字符集的设置 <br />           字符集和校对规则有四个默认级别的默认设置, 服务器级别, 数据库级, 表级和字段级, 分别在不同的地方设置作用也不相同 <br />           显示当前的服务器字符集和校对规则 <br />                SHOW VARIABLES LIKE 'character_set_server' <br />6.索引的设计和使用<br />      每个存储引擎支持每个表至少16个索引 <br />      只有 MyISAM支持 fulltext 索引, 并且只是支持 char, varchar, text列 <br />      设计原则 <br />           搜索的索引列不一定是要选择的列, 适合索引的列是 where后的列, 而不是出现在select后的列 <br />           使用唯一索引,考虑列的分部, 唯一值的列索引效果更好 <br />           使用短索引, 如果有一个char(200)列, 如果在前10个或20个字符内,多数值是唯一的, 那么就不要对整个列进行索引, 因为短索引能够节省大量的空间,所以查询更快. <br />           最左前缀 <br />           不要过度索引 <br />           考虑列上的比较类型,索引可用字符和between运算 <br />      BTree索引和hash索引 <br />      如何使用索引 <br />           索引用于快速找出在某个列上的特定值的行,如果不使用索引,mysql必须从第一行开始然后读完整个表直到找出相关的行 <br />           大多数索引(Primary Key, Unique, Index, fulltext) 使用btree, 只是空间列类型的索引使用R-tree, 并且Memory还支持hash索引 <br />7. 锁机制和事务控制<br />      加锁 <br />           Lock tables tbl_name <br />           unlock tables <br />           innodb提供 行级锁, 支持共享锁和排他锁.
